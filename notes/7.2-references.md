# Module 7.2 – References

### Learning Goals

- Understand what a **reference** is in C++
- Learn how references differ from pointers
- Understand when references are used instead of pointers
- Learn how references simplify function interfaces
- Understand reference binding rules

---

## The Core Idea

A **reference** is an **alias for another variable**.

Once created, a reference behaves like the original variable.

Example:

```cpp
int x = 10;
int& r = x;
```

Now:

> `r` refers to `x`

Meaning:

> `x` and `r` refer to the same memory

If we modify `r`:

```cpp
r = 20;
```

Then:

```
x == 20
```

---

## Memory Picture

Example:

```cpp
int x = 10;
int& r = x;
```

Conceptually:

```
memory:

address A → 10

x → address A
r → address A
```

Both names refer to **the same object**.

---

## Reference Syntax

References use the `&` symbol.

Declaration:

```cpp
type& name = variable;
```

Example:

```cpp
double value = 3.14;
double& ref = value;
```

Important rule:

> references must be initialized

This is **illegal**:

```cpp
int& r; // error
```

Because a reference must always refer to something.

---

## References vs Pointers

In C we would write:

```c
int x = 10;
it *p = &x;
```

Access requires dereferencing:

```c
*p = 20;
```

In C++ references:

```cpp
int x = 10;
int& r = x;

r = 20;
```

**No dereferencing needed**.

---

## Key Differences

| Feature | Pointer | Reference |
|---------|---------|-----------|
| Can be null | yes | no |
| Must be initialized | no | yes |
| Reassignable | yes | no |
| Syntax | explicit dereference | direct access |

Example:

```cpp
int a = 5;
int b = 10;

int& r = a;
r = b;
```

This does **not** rebind the reference.

It assigns the value:

```
a = 10;
```

`r` still refers to `a`.

---

## References in Function Parameters

References are most useful for **function parameters**.

In C:

```c
void increment(int *x) {
    *(x)++;
}
```

Call:

```c
increment(&value);
```

In C++:

```cpp
void increment(int& x) {
    x++;
}
```

Call: 

```cpp
increment(value);
```

The function still modifies the original variable.

But the interface is cleaner.

---

## Example: Swap Function

C version:

```c
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}
```

Call:

```c
swap(&x, &y);
```

C++ version:

```cpp
void swap(int& a, int& b) {
    int temp = a;
    a = b;
    b = temp;
}
```

Call:

```cpp
swap(x, y);
```

Much easier to read.

---

## References cannot be NULL

Pointers can be `NULL`:

```c
int *p = NULL;
```

But references must always refer to a valid object.

This makes them safer.


---

## Const References

One of the most important patterns in C++:

```cpp
const Type& parameter
```

Example:

```cpp
void print(const std::string& s);
```

This means:

> read-only reference

Benefits:

- no copy
- no modification
- efficient

This pattern is **extremely common in C++ APIs**.

---

## When to use References

Use references when:

- object must exist
- object should not be `NULL`
- function modifies the argument
- or avoids copying large objects

Use pointers when:

- `NULL` is meaningful
- optional object
- dynamic memory structures

---

## References in Modern C++

References are used everywhere:

Examples:

- function parameters
- range-based loop
- operator overloads
- STL algorithms

Example:

```cpp
std::vector<int> vector = {1,2,3,4};

for (auto& x : vector) {
    x *= 2;
}
```

`x` is a reference to each element.

No copies are made.

---

## Mental Model for C Programmers

A good way to think about references:

> reference = automatically dereferenced pointer

We still get the same behavior, but the syntax is simpler.

---

## Common Beginner Mistake

Example:

```cpp
int x = 5;
int& r = x;

int y = 10;
r = y;
```

This does **not** make `r` refer to `y`.

Instead:

> `x` becomes 10

Because `r` remains bound to `x`.

---

## Takeaway

> - A **reference** is **an alias for another variable**.  
> - References must be **initialized**.  
> - References **cannot be `NULL`**.  
> - References **cannot be rebound**.  
> - References simplify function interfaces.  
> - `const Type&` is a common pattern for passing large objects efficiently.
