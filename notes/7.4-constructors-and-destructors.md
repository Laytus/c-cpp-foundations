# Module 7.4 – Constructors and Destructors

### Learning Goals

- Understand how C++ objects are **created and destroyed**
- Learn what **constructors** are and how they initialize objects
- Learn what **destructors** are and how they release resources
- Understand the lifecycle of objects
- See how constructors and destructors enable **RAII**

---

## The Core Idea

In C++, objects have a **lifecycle**:

- *object created* → constructor runs
- *object used*
- *object destroyed* → destructor runs

These functions are automatically called by the language.

We never call them manually.

---

## Constructors

A **constructor** is a special function that runs when an object is created.

It initializes the object.

Example:

```cpp
class Point {
public:
    int x;
    int y;

    Point(int a, int b) {
        x = a;
        y = b;
    }
};
```

Usage:

```cpp
Point p(3, 4);
```

What happens:

- memory allocated for `p`
- constructors runs
- `x = 3`
- `y = 4`

---

## Constructor Syntax

Constructor rules:

- same name as class
- no return type
- called automatically

Example:

```cpp
class Counter {
public:
    int value;

    Counter() {
        value = 0;
    }
};
```

Usage:

```cpp
Counter c;
```

`c.value` becomes `0`.

---

## Default Constructor

A **default constructor** takes *no arguments*.

In the previous example, the `Counter`'s constructor does not take any argument.

```cpp
Counter c;
```

---

## Multiple Constructors

C++ allows **constructor overloading**.

Example:

```cpp
class Point {
public:
    int x;
    int y;

    Point() {
        x = 0;
        y = 0;
    }

    Point(int a, int b) {
        x = a;
        y = b;
    }
};
```

Usage:

```cpp
Point a;       // (0, 0)
Point b(3, 4); // (3, 4)
```

---

## Destructors

A **destructor** runs when an *object is destroyed*.

Syntax:

```cpp
~ClassName()
```

Example:

```cpp
class Resource {
public:

    Resource() {
        std::cout << "acquired\n";
    }
    
    ~Resource() {
        std::cout << "released\n";
    }
};
```

Usage:

```cpp
void f() {
    Resource r;
}
```

Output:

```
acquired
released
```

The destructor runs automatically when `r` leaves scope.

---

## Object Lifetime and Scope

Example:

```cpp
void f() {
    Resource a;

    {
        Resource b;
    }
}
```

Execution order:

```
constructor a
constructor b
destructor b
destructor a
```

Object are destroyed in **reverse order of construction**.

This is called:

> **stack unwinding**

---

## Why Destructors Matter

Destructors enable automatic cleanup.

Example:

```cpp
class Buffer {
    int* data;

public:

    Buffer(size_t n) {
        data = new int[n];
    }

    ~Buffer() {
        delete[] data;
    }
};
```

Usage:

```cpp
void f() {
    Buffer buf(100);
}
```

When `buf` leaves scope:

```
delete[] automatically called
```

This is **RAII in practice**.

---

## Stack vs Heap objects

Stack object:

```cpp
Buffer buf(100);
```

Destructor runs automatically when the scope ends.

Heap object:

```cpp
Buffer* buf = new Buffer(100);
```

Now the destructor runs only if:

```cpp
delete buf;
```

This is why modern C++ avoids raw `new` when possible.

Instead we use RAII containers like:

```cpp
std::vector
std::unique_ptr
```

---

## Example with `std::vector`

```cpp
void f() {
    std::vector<int> v(1000);
}
```

Internally:

- constructor → allocate memory
- destructor → free memory

No manual cleanup needed.

---

## Constructor Initialization Lists

The recommended way to initialize members is:

```cpp
class Point {
public:
    int x;
    int y;

    Point(int a, int b) : x(a), y(b) {}
};
```

This initializes members **directly**, instead of assigning later.

This is more efficient.

---

## Lifecycle Summary

When we write:

```cpp
Point p(3, 4);
```

The sequence is:

- memory allocated
- constructor runs
- object used
- destructor runs
- memory released

All automatic.

---

## Common Beginner Mistakes

#### Forgetting destructors

Example:

```cpp
class Bad {
    int* data;

public:
    Bad() {
        data = new int[100];
    }
};
```

Memory leak.

Because destructor missing.

#### Using raw new

Example:

```cpp
Buffer* b = new Buffer(100);
```

Better modern style:

```cpp
std::vector<int> v(100);
```

or

```cpp
std::unique_ptr<Buffer> b = std::make_unique<Buffer>(100);
```

---

## Connection to C knowledge

In C we manually wrote:

```c
init_struct()
destroy_struct()
```

In C++ constructors and destructors formalize this pattern.

C pattern:

```c
init()
use()
destroy()
```

C++ pattern:

```
constructor
use
destructor
```

---

## Takeaway

> - **Constructors** *initialize objects*.  
> - **Destructors** *clean up resources*.  
> - They run automatically.  
> - Destructors enable RAII.  
> - Objects are destroyed in reverse order of construction.  
> - This makes resource management safer and deterministic.
