# Module 7.1 – C++ vs C: What Changes

### Learning Goal

- Understand how C++ builds on top C
- Learn the main philosophy differences between C and modern C++
- Understand why C++ introduces abstractions like RAII and containers
- Learn what problems C++ tries to solve compared to C
- Develop the correct mental model before learning the language features

---

## The Core Idea

C++ started as **"C with classes"**, but modern C++ is much more than that.

The key shift is this:

| C | Modern C++ |
|---|------------|
| Manual resource management | Automatic resource management |
| Raw pointers | Ownership types |
| Procedural style | Type-oriented design |
| Custom data structures | Standard library containers |
| Error codes | Exception / RAII patterns |

Modern C++ tries to make programs:

```
safer
more expressive
less error-prone
```

without sacrificing performance.

---

## The Philosophy Shift

THe most important difference between C and modern C++ is this:

#### C

Programmers manually manage everything.

Example: 

```
allocate memory
remember to free memory
handle errors manually
track ownership in your head
```

This gives full control but is **easy to get wrong**.

#### Modern C++

The language helps enforce correct behavior.

Example:

```
objects clean themselves up
containers manage memory
types express ownership
```

This dramatically reduces bugs.

---

## Example: Memory Mangement

#### C style

```c
int *arr = malloc(100 * sizeof(int));

if (!arr) return -1;

/* use arr */

free(arr);
```

Problems:

```
easy to forget free
possible memory leaks
ownership unclear
```

#### C++ style

```cpp
std::vector<int> arr(100);
```

The container:

- allocates memory
- manages resizing
- frees memory automatically

No manual cleanup required.

---

## Example: Resource Ownership

In C:

```
who owns this pointer?
```

This question is often unclear.

Example:

```c
char *buffer = create_buffer();
```

Who frees it?

```
caller?
callee?
shared?
```

In modern C++:

Types express ownership.

Example:

```cpp
std::unique_ptr<Buffer> buf;
```

Meaning:

```
this object owns the resource exclusively
```

Ownership becomes explicit.

---

## RAII – The Central Idea of C++

One concpet explains much of modern C++:

#### RAII

> Resource Acquisition Is Initialization

Meaning:

> resources are tied to object lifetime

Example:

```cpp
std::vector<int> v(10);
```

Memory is:

```
allocated in constructor
freed in destructor
```

This prevents:

- memory leaks
- forgotten cleanup
- inconsistent states

---

## The Standard Library

C relies heavily on custom implementations.

Example:

```
linked lists
dynamic arrays
strings
hash tables
```

C++ provides these in the **Standard Template Library (STL)**.

Examples:

| C++ Type | Purpose |
|----------|---------|
| `std::vector` | dynamic array |
| `std::string` | string handling |
| `std::map` | ordered dictionary |
| `std::unordered_map` | hash table |

This saves enormous development time.

---

## Performance

A common myth:

> C++ is slower than C

Modern C++ is typically **just as fast as C**, because:

- abstraction compile away
- templates generate optimized code
- containers use efficient implementations

This principle is called:

> zero-cost abstractions

Meaning:

> you only pay for what you use

---

## What we will still use from C

The C knowledge remains extremely valuable.

We will still use:

- pointers
- struct layout
- memory model
- stack vs heap
- manual control when needed

Many C++ libraries internally rely on these concepts.

The C background is a **huge advantage**.

---

## What we will use less

In modern C++, we usually avoid:

- `malloc` / `free`
- raw arrays
- manual memory ownership
- C-style strings

These are replaced by safer abstraction.

---

## Mental Model for the Transition

A useful way to think about modern C++:

> C + stronger types + automatic resource management

Or even:

> C with guardrails

We still get full control when needed, but the language helps prevent common mistakes.

---

## Takeaway

> - C++ builds on C but introduces *safer abstractions*.  
> - Modern C++ focuses on **resource safety and expressiveness**.  
> - Containers replace manual memory management.  
> - Ownership is expressed through types.  
> - RAII ties resource lifetime to object lifetime.  
> - Performance remains comparable to C.
